## Key Decisions Made

### 1. Architecture Choice
- **Decision**: Move from traditional service to AI agent-based architecture with **Supervisor Agent**
- **Reason**: Better for open source adoption, extensible, intelligent decision making, true modularity
- **Implementation**: Multi-agent system with intelligent supervisor pattern

### 2. Supervisor Agent Pattern
- **Decision**: Implement true supervisor agent instead of static orchestrator
- **Benefits**: Dynamic workflow planning, runtime adaptation, load balancing, error recovery
- **Intelligence**: Uses Claude to make workflow decisions and handle failures

### 3. Knowledge Base Integration
- **Decision**: Use instrument documentation as knowledge base for deterministic converter generation
- **Reason**: Team suggestion to make converters more accurate and scientifically precise
- **Benefits**: Deterministic vs probabilistic generation, exact field mappings, proper calibrations
- **Implementation**: Knowledge Base Agent + Enhanced Converter Generation Agent

### 4. Strands Framework Integration
- **Decision**: Use Strands agents framework instead of custom agent implementation
- **Reason**: Industry best practices, proper tool integration, extended thinking capabilities
- **Benefits**: Better agent coordination, structured interfaces, enhanced reasoning
- **Implementation**: Refactored all agents to use Strands patterns with @tool decorators

### 5. Allotrope Foundation Integration
- **Decision**: Integrate with official Allotrope GitLab repository for standards compliance
- **Repository**: `https://gitlab.com/allotrope-public/asm.git`
- **Benefits**: Regulatory compliance, official templates, schema validation
- **Implementation**: Allotrope Standards Agent with daily synchronization

### 6. **🆕 Simple POC Deployment**
- **Decision**: Deploy minimal working POC for customer demonstration
- **Reason**: Customer meeting alignment - need working demo quickly
- **Benefits**: Proves AI-powered conversion concept, ready for testing
- **Implementation**: CDK stack with Lambda + API Gateway + Bedrock Claude

### 7. Technology Stack
- **AI Models**: AWS Bedrock Claude 3.7 Sonnet (via Strands BedrockModel + direct Lambda)
- **Agent Framework**: Strands Agents with strands-agents-tools (future full implementation)
- **Language**: Python for agents, TypeScript/Python for generated converters
- **Deployment**: **🚀 LIVE on AWS** - Container-based for future scaling
- **Knowledge Storage**: S3 bucket with instrument documentation (planned)
