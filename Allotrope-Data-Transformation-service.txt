AWS-Allotrope scientific data transformation service 
Opportunity and Context 

ASM (Allotrope Simple Model) serves as a standardized format for capturing lab data making it easier to exchange scientific data across systems. 
Allotrope and AWS are partnering to enable AI ready scientific data at scale by creating a set of vendor neutral services in the cloud that lower the barrier of entry to adoption of ASM.  These services will 
* Eliminate undifferentiated need for to develop the conversion tools and set up infrastructure to run the tools
* Lower cost of adoption by leveraging shared infrastructure[TD1][TL2], governance and maintenance across organizations
* Maintain a shared collaborative library of mappings from vendors specific source to the standardized ASM format 
* Ensure data security and confidentiality with no persistent data storage
* Help manage lifecycle of Allotrope standards as they evolve
The foundational services required are
ASM Transformation as a Service  (ATaaS) : This will be a cloud based service accessible via an API to enable transform raw data acquired from an instrument or a software system into an ASM file including metadata standardization, handling organization-specific fields, and integrating reference data from multiple sources into ASM files.
ASM data validation as a service (DVaaS): This will be a certification service that validates  that any ASM output created by users or third-party solution conforms to a common understanding of structures and constraints defined in the ASM standard. 
Additional services: Once the foundational services for creating normalized data are established, we will be able to create additional downstream services such as
- Create knowledge graph that helps mine the scientific data available in ASM format
- Create a methods repository that leverages ASM template to share and standardize commonly used methods within an organization or across organizations working in partnership
As a next step, we are looking to execute a Proof of Concept for foundational services in consultation with Merck to define the scope of service, establish a solution approach and define criteria for viability of these services.

Proposed POC

Use Case Selection. Merck/Allotrope are prioritizing use cases that address the most significant current barriers to adoption. Additional use cases will be incorporated in future phases.
Scope Definition. Merck will identify proof-of-concept (POC) scopes that demonstrate substantial business value.
Architecture Development. AWS will develop a reference architecture and prototype following AWS best practices.
Initial Deployment Model. The initial solution will be a self-managed AWS architecture deployed in the customer's account (Merck or other Allotrope member). Deployment will be automated via cloud templates, providing end users with a fully managed service experience.
Future Evolution. Based on adoption / consumption metrics, development feedback, and customer needs, we will evaluate progression toward a full AWS managed service in later phases.

Proposed Solution 

The Allotrope Converter Service represents a modern approach to laboratory data standardization, designed to meet the demanding requirements of pharmaceutical research environments.  This service enables scientists to convert instrument data into standardized formats without requiring local software installation or persistent storage of sensitive data in the cloud.
How it Works:
Data Generation and Submission
When a laboratory instrument generates data, the file is automatically routed through a secure edge gateway. This gateway serves as the bridge between the laboratory environment and the cloud-based conversion service. Scientists can also manually submit files through a secure interface when needed.
Intelligent Processing:
Upon receiving a file, the service:
1. Identifies the source instrument and file format
2. Checks file and route to one of the following processing strategies:. [TL3][TL4]
a. Selects the right approved converter from the library. If there is a substantive change (like new version available) route for approval. 
b. Generates new conversion code. Routes for scientific review and approval.
3. Executes the converter
4. Generates compliant ASM file output
5. Returns the converted file to the requestor

Scientists Experience:[TL5]
From the scientist’s perspective, the process is straightforward:
* Generate data from laboratory instrument
* Automatic submission through edge gateway
* Receive converted ASM file
* Utilize standardized data
* Visualize data in a knowledge graph

Data Admin Experience
* Get notified of new converter
* Review converter code and quality report
* Approve, edit or reject converter

Technical Solution Architecture
The Allotrope Converter Service provides a cloud-based, transient data conversion capability that transforms laboratory instrument data into standardized Allotrope Simple Model (ASM) format. The service operates on a request-response model, processing data without persistent storage of source files or converted outputs in the cloud environment.
System Overview
