# How the Nova FLEX2 Conversion Works

## Question: How did the system know it was Nova FLEX2?

**Answer: It DIDN'T automatically detect it.**

The system does NOT have automatic instrument detection. We manually created a custom converter specifically for Nova FLEX2 based on customer information.

---

## What Information Comes From Where?

### 1. FROM THE CSV FILE (Data Values)
```
CSV Column          → ASM Field
-----------------------------------------
Sample ID           → sample_identifier
pH = 7.183          → pH.value = 7.183
Operator            → analyst
Gln = 1.80          → glutamine concentration
Date & Time         → measurement_time
```

### 2. HARDCODED IN THE CONVERTER (Metadata)
```
NOT in CSV          → Added by Converter
-----------------------------------------
<no instrument>     → device_identifier = "bioprofile flex2"
<no manufacturer>   → product_manufacturer = "nova biomedical"
<no device type>    → device_type = "solution analyzer"
<no manifest>       → $asm.manifest = "http://purl.allotrope.org/.../solution-analyzer.manifest"
<no IDs>            → measurement_identifier = UUID (generated)
```

### 3. DERIVED/TRANSFORMED (Logic)
```
CSV Value           → ASM Transformation
-----------------------------------------
"Gln"               → "glutamine" (full chemical name)
"1.80"              → {value: 1.8, unit: "mmol/L"} (added unit)
"1:2"               → 0.5 (calculated 1/2)
"11/1/2025 4:46 AM" → "2025-11-01T04:46:26.000+00:00" (ISO8601)
```

## How Was the Converter Created?

### Step 1: Customer Provided Context
- File name: `SampleResults2025-November.csv`
- Instrument: Nova BioProfile FLEX2
- Purpose: Solution analyzer for cell culture monitoring

### Step 2: Analyzed CSV Structure
```csv
"Date & Time","Sample ID","pH","PO2","PCO2","Gln","Glu","Gluc","Lac",...
```

Identified:
- Blood gas measurements (pH, PO2, PCO2)
- Metabolites (Gln, Glu, Gluc, Lac, NH4+, Na+, K+, Ca++)
- Osmolality (Osm)
- Metadata (operator, lot numbers, timestamps)

### Step 3: Mapped to ASM Schema
Chose: `solution-analyzer/REC/2025/06/solution-analyzer.manifest`

Created measurement types:
- Blood gas analyzer (PO2, PCO2, O2/CO2 saturation)
- pH sensor (pH, temperature)
- Osmolality sensor (Osm)
- Metabolite analyzer (all analytes)

### Step 4: Hardcoded Knowledge
```python
# Device information (NOT in CSV)
device_identifier = "bioprofile flex2"
product_manufacturer = "nova biomedical"

# Column name → Chemical name mapping
"Gln" → "glutamine"
"Glu" → "glutamate"
"NH4+" → "ammonium"

# Column name → Unit mapping
"Gln" → "mmol/L"
"Gluc" → "g/L"
"Osm" → "mosm/kg"
```

## Why Doesn't Multi-Instrument Service Work?

The allotropy library has a `NOVABIO_FLEX2` vendor, but it expects a different file format than what Merck provides. The customer's CSV has:
- Custom column names
- Specific formatting
- Additional metadata fields

So we needed a **custom converter** tailored to Merck's exact format.

---

## Summary

| Information Type | Source |
|-----------------|--------|
| **Data values** (pH=7.183, Sample ID, etc.) | FROM CSV |
| **Device info** (manufacturer, model) | HARDCODED in converter |
| **Chemical names** (glutamine, not Gln) | HARDCODED mapping |
| **Units** (mmol/L, g/L) | HARDCODED knowledge |
| **ASM structure** (manifest, schema) | HARDCODED template |
| **UUIDs, timestamps** | GENERATED by converter |

**The converter is essentially a translation template** that knows:
1. What the CSV columns mean
2. What instrument produced them
3. How to structure them in ASM format
4. What metadata to add

It does NOT use AI or automatic detection - it's a hand-coded Python script based on domain knowledge.
