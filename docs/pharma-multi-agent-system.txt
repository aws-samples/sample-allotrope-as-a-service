## Complete Multi-Agent System for Pharmaceutical Data Conversion

I've designed a **production-ready multi-agent system** using:

### Architecture Highlights

The system uses a **supervisor agent** that coordinates four specialized sub-agents:

1. **Parser Agents** (Agilent, Waters, Thermo) - Handle vendor-specific instrument formats
2. **Transformer Agent** - Converts parsed data to ASM format
3. **Validator Agent** - Validates against ASM schemas with intelligent error recovery
4. **Storage Integration** - S3 for input/output file management

### Key Advantages Over Traditional Python Script

**Intelligent Error Handling**: Agents can reason about validation failures and suggest corrections automatically, rather than just failing with error messages.

**Adaptive Processing**: The supervisor dynamically routes to the appropriate parser based on instrument type detection, and can handle new formats by updating agent instructions rather than rewriting code.

**Self-Healing**: If validation fails, the validator agent can analyze the errors and work with the transformer agent to fix common issues automatically.

**Natural Scalability**: Deploy on ECS Fargate with auto-scaling based on workload - the agent framework handles parallel processing naturally.

**Easier Maintenance**: Update agent behavior by modifying instructions rather than refactoring code, making it easier for your team to adapt to new ASM schema versions.

### Quick Start

```bash
# Build and run locally
docker build -t pharma-asm-converter .
docker run -p 8080:8080 \
  -e AWS_REGION=us-east-1 \
  -e BEDROCK_MODEL_ID=anthropic.claude-3-5-sonnet-20241022-v2:0 \
  pharma-asm-converter

# Convert a file
curl -X POST http://localhost:8080/convert \
  -H "Content-Type: application/json" \
  -d '{
    "input_file": "s3://bucket/sample.xml",
    "technique_type": "HPLC",
    "output_bucket": "output-bucket",
    "output_key": "converted/sample_asm.json"
  }'
```
