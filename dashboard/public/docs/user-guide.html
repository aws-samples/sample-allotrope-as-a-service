Overview

The ASM Transformation Service converts laboratory instrument output files to Allotrope Simple Model (ASM) format with built-in validation and data integrity verification.

Getting Started

Step 1: Create an Instrument Config

Go to the Instrument Config Creator tab. Select your instrument from the dropdown or choose "Custom Instrument" for unlisted instruments. Fill in your instrument details (serial number, software version, location, source file path, timezone). Download the generated JSON file. You only need to do this once per instrument.

Step 2: Convert an Instrument File

Go to the Convert Instrument File tab. Upload your raw instrument output file and the instrument config JSON. Click "Convert to ASM & Validate". The service will convert your file, validate the ASM output, and show a data integrity verification report.

Step 3: Review Results

After conversion you will see:

  • Validation status — VALID or INVALID with error and warning counts
  • Data Integrity Verification — field-by-field proof that source values were preserved
  • Download options — ASM file (JSON) and validation report (PDF)

Validating an Existing ASM File

If you already have an ASM file and want to check compliance, use the Validate ASM File tab. Upload your ASM JSON file and click "Validate ASM File". The service checks schema compliance, required fields, data traceability, and provides recommendations with side-by-side comparisons showing what to fix.

Registering a Custom Converter

There are two ways to create a custom converter:

Option A: Build Manually

  1. Build a converter following the Custom Converter Requirements
  2. Go to the Converter Management tab
  3. Click "Register New Converter" and upload your Python file
  4. A reviewer will approve or reject the converter
  5. Once approved, the converter is automatically used for matching instruments

Option B: Generate with AI

  1. Go to the AI Converter Generator tab
  2. Enter your instrument details (vendor, model, type)
  3. Upload a sample instrument file (only headers + a few rows are sent to AI)
  4. Click "Generate Converter with AI" — Bedrock Claude generates the code
  5. The converter is auto-registered as PENDING in Converter Management
  6. Review the generated code, test it, then approve for production use

Data Integrity Verification

When a converter supports it, the service provides field-by-field proof that every value from the source instrument file was preserved exactly in the ASM output. This is critical for FDA 21 CFR Part 11 and EMA Annex 11 compliance.

The integrity report shows: source field name, source value, ASM field name, ASM value, unit, and match status for every mapped field.

Custom Validation Rules

The service supports pluggable validation rule sets that add checks beyond the Allotrope JSON schema. Use the Validation Rules tab to:

  • Start from template — load 18 pre-built rules based on Allotrope building principles
  • Import rules — upload a JSON rule set file
  • Build rules — create individual rules using the form-based builder
  • Enable/disable — toggle rule sets on or off

Enabled rule sets are automatically applied when you validate or convert files. Each finding in the results is tagged with which rule set caught it.

Rule sets are organized by level: L1 (supplementary schema checks), L3 (Allotrope building principles), L4 (organization-specific). L2 (Data Integrity Verification) runs automatically via the converter's field_mapping.

Onboarding a New Instrument

To onboard a new instrument: build or generate a converter, register it in Converter Management, assign it in the Instrument Registry, create an instrument config, and the instrument is ready for conversion on every future use.

Selecting a Specific Converter

When multiple converters are registered for the same instrument, you can choose which one to use:

  • In the instrument config file — set the converter_id field for automated pipelines
  • On the Convert tab — use the Converter Selection dropdown to override on the fly

Priority: screen selection > config file > auto-select. If neither specifies a converter, the system picks the best available automatically.
